About the Data

48 closed-loop braking simulations generated in MATLAB/Simulink across 8 road scenarios, 3 brake force distributions, and 2 controller modes. Each simulation runs until the vehicle stops, producing time-series of slip, friction, and ESC signals.

Preprocessing pipeline
Raw 765k → ABS filter (abs_active=1) → 479k
→ Confidence filter (α > 0.01) → 340k

→ Balance Snow≤12k, Dry/Wet≤6k → 20,616 rows
Key signals per row
slip — wheel slip ratio (encoder)
mu_noisy — friction coefficient (force estimation)
g_esc — ESC gradient from LPF demodulator
a_esc — ESC dither amplitude
alpha — identifier confidence (0–1)
surface — ground truth label (Dry / Wet / Snow)
Algorithm
Three-Layer ESC Identifier

Three layers fire in cascade — each activates when its precondition is met. The system always returns a best estimate, even during warmup.

Layer 1 — Always active
Lookup Table
Maps observed peak friction μ_max to a coarse c₂ estimate via linear interpolation. Immediate, fallback-safe.
μ_max → c₂_coarse
→
Layer 2 — ESC gradient available
ESC-Gradient Analytic
Converts ESC demodulator output to true friction gradient, then solves analytically for c₁ and c₃ at any operating point — not just the peak.
g_esc → c₁, c₃
→
Layer 3 — Sufficient slip excitation
Brent Root-Finding
When ABS excitation places the wheel far enough from the peak, solves for c₂ from two curve points using Brent's method (guaranteed convergence).
|s_probe − s_peak| > δ → c₂_fine
Burckhardt Friction Model
μ(s) = c₁ · (1 − e−c₂|s|) − c₃ · |s|
c₁ — friction amplitude  ·  c₂ — curve sharpness (key discriminant: 24 dry / 34 wet / 94 snow)  ·  c₃ — slope after peak

Zero added noise. The mu_noisy column already includes realistic ABS sensor noise. Classifier achieves perfect F1 on the prepared dataset — this is the clean baseline.
Why F1 macro + Balanced Accuracy? With equal batch counts per surface, F1 macro = BAC — neither inflates due to class imbalance. F1 is standard in ML; BAC is more readable to automotive engineers.
